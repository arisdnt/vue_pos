=============================================
KEY PATTERNS:
=============================================

1. MODAL-BASED CRUD
   - List pages (pages/) trigger popups
   - All forms in popups/ directory
   - No separate create/edit routes

2. SINGLE RESPONSIBILITY (Max 200 LOC)
   - Each component has one clear purpose
   - Composables handle business logic
   - Components only handle presentation

3. WINDOWS 10 NATIVE FEEL
   - No rounded corners
   - Square, clean design
   - Efficient space usage
   - Professional business aesthetic

4. OFFLINE-FIRST
   - Dexie (IndexedDB) for local storage
   - Supabase for cloud sync
   - Outbox pattern for queue management

5. MULTI-STORE SUPPORT
   - Owner: tidak terikat store (all access)
   - Manager/Cashier: terikat store tertentu
   - store_id filter otomatis via RLS

=============================================
COMPONENT NAMING CONVENTION:
=============================================

- Base components: Win{Component}.vue
- Popups: {Feature}{Action}Popup.vue
- Pages: {Feature}ListPage.vue / {Feature}Page.vue
- Composables: use{Feature}.ts
- Stores: {feature}Store.ts
